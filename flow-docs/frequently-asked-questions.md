# <a name="frequently-asked-questions"></a>Veelgestelde vragen

## <a name="audience-and-strategy"></a>Doel groep en-strategie
### <a name="what-is-microsoft-flow"></a>Wat is Microsoft Flow?
Microsoft Flow is een Cloud service waarmee line-of-Business-gebruikers praktisch en eenvoudig werk stromen kunnen bouwen waarmee tijdrovende zakelijke taken en processen in toepassingen en services worden geautomatiseerd.

### <a name="who-is-the-intended-audience-for-microsoft-flow"></a>Wie is de doel groep voor Microsoft Flow?
Microsoft Flow heeft twee verschillende doel groepen:

* Line-of-Business "burger integrators" in bedrijfs organisaties die hieraan samen werken om de verantwoordelijkheid voor zakelijke oplossingen dichter bij het bedrijf zelf te zetten.
* IT-besluit vormers die in het bezit zijn van line-of-Business-partners om hun eigen oplossingen te maken, zodat IT-professionals en integratie specialisten hun expertise kunnen concentreren op meer geavanceerde integratie hulpprogramma's, zoals Azure Logic Apps.

### <a name="how-do-microsoft-flow-and-logic-apps-relate-to-each-other"></a>Hoe kunnen Microsoft Flow en Logic Apps betrekking hebben op elkaar?
Microsoft Flow biedt functies waarmee line-of-Business-gebruikers geautomatiseerde werk stromen kunnen maken. Logic Apps is een Azure-service die dezelfde fantastische functies van Microsoft Flow biedt, plus functies zoals integratie met Azure Resource Manager en Azure Portal, Power shell en xPlat CLI, Visual Studio en aanvullende connectors. Meer [informatie over Logic apps](https://azure.microsoft.com/services/app-service/logic/).

### <a name="how-does-microsoft-flow-fit-in-microsofts-overall-business-application-platform-strategy"></a>Hoe past Microsoft Flow in de algehele Business Application Platform strategie van micro soft?
Microsoft Flow maakt deel uit van een krachtige en aanpas bare Business Application Platform die PowerApps, de Common Data Service, Dynamics 365 en Office 365 bevat. Op basis van dit platform kunnen onze klanten, onze partners en onze ISV-partners oplossingen maken voor hun eigen bedrijven, hun branche, voor functionele rollen of zelfs voor specifieke geografische regio's. Line-of-Business-gebruikers die hun bedrijfs behoeften beter begrijpen, kunnen nu eenvoudig gegevens en processen analyseren, opstellen en stroom lijnen. Professionele ontwikkel aars kunnen de line-of-Business-toepassing voor automatisering, analyses en apps eenvoudig uitbreiden om gebruik te maken van Azure-Services zoals functions, App Service en Logic Apps. API-connectors, gateways en de micro soft-Common Data Service maken het mogelijk om meer waarde te verkrijgen van services of gegevens die al in gebruik zijn, hetzij in de Cloud of on-premises.

## <a name="functionality"></a>Voorziening
### <a name="what-do-i-need-to-use-microsoft-flow"></a>Wat moet ik Microsoft Flow gebruiken?
Als u Microsoft Flow wilt gebruiken, hebt u alleen een webbrowser en een e-mail adres nodig.

### <a name="what-browsers-and-devices-can-i-use-with-microsoft-flow"></a>Welke browsers en apparaten kan ik gebruiken met Microsoft Flow?

U kunt Microsoft Flow uitvoeren op alle moderne apparaten en browsers.

#### <a name="supported-devices"></a>Ondersteunde apparaten

Microsoft Flow is geweldig op moderne apparaten. Als u Microsoft Flow vanaf een mobiel apparaat wilt beheren, probeert u de Microsoft Flow mobiele app die beschikbaar is op [iPhone](https://itunes.apple.com/app/microsoft-flow/id1094928825?ls=1&mt=8), [Android](https://play.google.com/store/apps/details?id=com.microsoft.flow)en [Windows Phone](https://www.microsoft.com/p/microsoft-flow/9nkn0p5l9n84?rtc=1#activetab=pivot:overviewtab).

#### <a name="supported-browsers"></a>Ondersteunde browsers

U kunt het beste de meest recente browser gebruiken die compatibel is met uw besturings systeem. De volgende browsers worden ondersteund:

* Micro soft Edge
* Internet Explorer 11
* Safari
* Chrome
* Firefox

### <a name="which-email-addresses-are-supported"></a>Welke e-mail adressen worden ondersteund?
Microsoft Flow ondersteunt e-mail adressen die eindigen met iets behalve. gov en. mil.  

### <a name="is-microsoft-flow-available-on-premises"></a>Is Microsoft Flow on-premises beschikbaar?
Microsoft Flow is alleen een open bare Cloud service. U kunt echter veilig verbinding maken met uw eigen on-premises Services via de on-premises gegevens gateway.

### <a name="what-services-can-microsoft-flow-connect-to"></a>Met welke services kan Microsoft Flow verbinding maken?
Microsoft Flow maakt verbinding met meer dan 100 gegevens bronnen uit het vak en we voegen meer tijd toe. Enkele voor beelden van gegevens bronnen en-services zijn:

* Point
* Dynamics 365
* OneDrive
* OneDrive voor bedrijven
* Google Drive
* Google-spread sheets
* Trello
* Twitter
* Keuzelijst
* Facebook
* SalesForce.com
* MailChimp
* Api's voor klanten

U kunt [hier](https://go.microsoft.com/fwlink/?LinkId=832211)een volledige lijst met beschik bare connectors vinden.

U kunt toegang krijgen tot gegevens bronnen in uw eigen IT-infra structuur via de [on-premises gegevens gateway](gateway-manage.md).

### <a name="what-are-templates"></a>Wat zijn sjablonen?
Sjablonen zijn vooraf gemaakte stromen voor populaire en algemene scenario's. Als u een sjabloon wilt gebruiken, hebt u alleen toegang tot de services in de sjabloon nodig en kunt u de vereiste instellingen invullen.

### <a name="what-data-sources-will-i-be-able-to-connect-to"></a>Met welke gegevens bronnen kan ik verbinding maken?
U kunt verbinding maken met meer dan 100 standaard services van micro soft en van derden, zoals Office 365, Twitter, share point, OneDrive, Dropbox, SQL Server en meer. U kunt ook verbinding maken met Premium Services zoals Sales Force en de Common Data Service voor PowerApps.

### <a name="how-do-i-connect-to-a-rest-api-in-my-flow"></a>Hoe kan ik verbinding maken met een REST API in mijn stroom?
U kunt verbinding maken met elke REST API die gebruikmaakt van JSON en ondersteuning biedt voor ten minste één van de meer dan tien verificatie methoden door [een aangepaste connector](developer/register-custom-api.md)te maken.

### <a name="how-do-i-connect-to-sql-server-and-other-on-premises-data-sources"></a>Hoe kan ik verbinding maken met SQL Server en andere on-premises gegevens bronnen?
U kunt verbinding maken met Services op uw lokale netwerk met behulp [van de on-premises gegevens gateway](gateway-manage.md).

### <a name="can-i-share-the-flows-i-create"></a>Kan ik de stromen delen die ik maak?
U kunt stromen op een van de volgende manieren delen:

* U kunt mede werkers of groepen in uw organisatie als eigen aren toevoegen aan uw stromen, zodat ze de stroom ook kunnen bewerken en beheren.
* Voor stromen die hand matig kunnen worden uitgevoerd, kunt u ook andere personen of groepen in uw organisatie machtigen om de stroom alleen uit te voeren.

### <a name="how-many-flows-can-i-have"></a>Hoeveel stromen kan ik hebben?
Microsoft Flow wordt geleverd met Maxi maal 50 stromen. Als u meer nodig hebt, kunt u deze aanvragen.

### <a name="can-flows-be-turned-off-or-disabled"></a>Kunnen stromen worden uitgeschakeld of uitgeschakeld?

Ja, elke stroom heeft een aan/uit-switch waarmee u de stroom van het verwerken van aanvragen kunt stoppen.

Raadpleeg de volgende tabel om te begrijpen hoe uw stroom reageert wanneer deze weer wordt ingeschakeld.

Trigger type|Beschrijvingen
-------|--------
Polling, zoals de **terugkeer patroon** trigger|Wanneer de stroom weer wordt ingeschakeld, worden alle niet-verwerkte gebeurtenissen verwerkt. Verwijder de stroom als u geen items wilt verwerken die in behandeling zijn.
Webhook|Wanneer de stroom weer wordt ingeschakeld, worden alleen nieuwe gebeurtenissen verwerkt die worden gegenereerd nadat de stroom is ingeschakeld.

### <a name="how-does-microsoft-flow-compare-to-sharepoint-designer-2013"></a>Hoe werkt Microsoft Flow vergelijken met share point Designer 2013?
Microsoft Flow is de opvolger van share point Designer voor veel algemene bedrijfs scenario's zoals goed keuringen, document revisie en onboarding/offboarding. Het is het standaard hulp middel voor het maken van zakelijke automatisering in share point vooruit.

### <a name="how-does-microsoft-flow-ensure-that-corporate-data-isnt-accidentally-released-to-social-media-services"></a>Hoe zorgt Microsoft Flow ervoor dat bedrijfs gegevens niet per ongeluk worden vrijgegeven aan sociale media services?
Beheerders kunnen [beleid voor preventie van gegevens verlies](prevent-data-loss.md) maken om ervoor te zorgen dat alleen goedgekeurde services worden gebruikt in Microsoft flow.

### <a name="does-microsoft-flow-support-service-accounts"></a>Ondersteunt Microsoft Flow service accounts?

Hoewel u stromen kunt maken met een service account, wordt het niet aanbevolen dit te doen als de referenties voor het service account worden gedeeld.
